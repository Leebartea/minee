Variables is highly important in programming, without it our data will be lost. Let's take for an example by using another "js" keyword which is "prompt".

    Using variables

    Variable store data and it can be use to retrieve any data stored. Example:

      var yourName = "Ola";

      In the above code, "var" keyword which is shorten for "variable" is used to create our variable which we named "yourName". This is then committed to the memory and can be retrieve for use. In the chrome console, we can just type the "yourName" and it'll give us the value it contains which is "ola". "yourName" is like a label on the data which the data is "ola" in this case. The "var" is a keyword just as "alert and prompt". It recognize that we're creating a new data container. And the container is label the name that follows the "var", which in this case is "yourName". We then set the variable to be equal to a value which in the case is "Ola". Now, every time we refer to "yourName", the computers knows we're referring to the string "ola". So basically, whenever we start with keyword "var", the computer knows is job is to help us create a box which we'll then label with any name that comes after the keyword "var". So after the equal signs lies the string that we place inside the box.

      Assigning variable new value

      The full meaning of "var" is variable, which means the data place inside it can be alter. For example;

      yourName = "Olatunji";

So when we decide to use the variable or when we decide to change the variable, we don't need to use the keyword "var" anymore.

    Basic Arithmetic & Modulo Operator

    In js, we can add, subtract, divide and so on. For examples:

    var add = 5 + 2;
    var sub = 5 - 2;
    var mul = 5 * 3;
    var mod = 5 % 3;

    The addition gives back "7", subtractions gives "3", multiplications gives 15 while the modulo gives back the remaining number which in this case is "2". The modulo is the weird one among all of them as it only gives back the remainder of the number we use it on. We use the modulo operator to show whether a number is "even or odd". e.g 45 % 2. If we get back 0, the number is "even", if the modulo end up as not equal to 0 then we have "odd" number.

      Working With Expressions

      var value = 5 + 6 * 2

      The expression follows the "BODMAS OR PEMDAS" rule. It multiply first then add. The above expression will return "17". This follow a maths rule that says "We must first multiply and divide before we add or subtract." This works exactly the same way in programming. We can get different value we want if we add parentheses to prioritize what should be done first. E.g;

      var value = (5 + 6) * 2

      This first operate on the numbers inside the bracket before multiplication. Which result to "22".

      Note; Despite knowing the order of operations, it's always best to add parentheses around numbers to make it distinctively clear. E.g;

      var value = 5 + (6 * 2);

      This makes the order be visible and clear enough.

    Increment & Decrement Expressions

    We've seen operators of normal maths. Here, we'll be expose with one that exists only in programming. For example;

    var x = 2;
    x = x + 1;

    The above will return back "3" as the value "x"

      Increment

      In programming, there is a better way to make an increment work.

      var x = 2;
      x ++;

      In most programming "x + 1" is same as "x++". The above code result to "3" as the result as well.

      Decrement

      The decrement also works in similar way.

      var x = 2;
      x --;

      "x - 1" is same as "x--". The above code result to "1" as the result as well. So the previous value of "x" which is 2 will be minus by 1. This is the decrement expressions.

      Increment To More Than One

      var x = 2;
      x += 3;

      "x + 3" is same as "x += 3". The above code result to "5" as the result. Other operator works same way.

      Increment By Value Of Variable

      var x = 2;
      var y = 3; x += y;

      "x += y" is same things as "x + 3". That is the value of previous x which is 2 plus the value of "y" which is 3. The above result to "5". This also work with others operators.

    Without functions, coding will be a complicated task to do. Functions reduce the workload in coding. For example;

    alert("Stand-up");
    alert("Move Left");

    alert("Sit Down");

    The above codes will have to run for every time we need to perform the specific type of task. Which is boring. Writing every line of the code repeatedly anytime we want to perform such task.

      Creating And Calling Functions

      Instead of going the painful way of repeating code, we can simply package the above set of instructions into a single package that we'll attach a name to. Functions allows us to create a series of instructions, package it into a block of code by adding curly braces as the boundary. For example;

      function commandButton() {
      alert("Stand-up");
      alert("Move Left");
      alert("Move Right");
      alert("Sit Down");
      }

      We've successfully package the instructions under a single package known as functions which we've given a name. Which the name in this case is "commandButton". We first indicate we're creating a function by first writing the keyword function, follow by the name then a set of parentheses. Then follow by curly bracket and the block of codes that lies between the curly brackets that we want to executed. So next time we want to run the code. We simply call the function. For example;

      commandButton();

      We simply type the name with the bracket to call the function which save of stress of having to type all the code out again.

        Notes

        The position inside the html document where we place the script tag matters a lot. With Css tag we place it inside the head element tag so that it get loaded first before any of the content. of our website which tends to be inside the body tag. This is highly necessary because, if we place the Css Link Tag inside the body tag instead, what happens as a result is that, our website will behave weirdly because it do read the code line by line from the top, so will have to get to the body tag before it execute our Css code which is not what we want.
        With Js, it is quite opposite because if inside our script we tried to change an html element that does not yet exist, then it will fail. So for example, inside our index.js we write a code to change html element document.querySelector("h1").innerHtml = "Good Bye"; This simply means, we want our code to look inside our document and the look for selector named h1 and when it finds the element, we then change the innerHtml which is the text that is inside the h1. Currently, it's set to Hello and we want to change it to Good Bye.
        Now, if the script we want to change the h1 is below the h1 that is after the h1 has been created, then the code will work as it'll find the innerHtml also known as the text. and change it as require. But if we move the script into the head tag section, then it'll fail because running from top to bottom will prevent it from seeing the h1 element we wanted to change. So it will fail before reaching where our h1 is.
        So best practice is always to bind the script tag inside the body tag. Added benefit of this is that, it add runtime speed to our website because it require more time to run the script, so placing it at the end makes our website loaded up fast and by the time it get to the Js zone, Js would be well ready.
